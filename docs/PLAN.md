## 一、总体目标

构建一个基于 `Python + FastAPI + SQLite + Vue` 的前后端分离个人博客系统，包含：

前台博客站点：首页导航、搜索、热门文章、最新文章列表、分页、文章详情、评论系统、footer 站点信息。

后台管理系统：管理员登录、角色权限控制、文章管理、审核流、分类标签、Markdown 编辑、媒体库、用户管理、评论审核。

## 二、总体架构

### 1. 仓库结构方向

当前仓库中仅有后端最小入口，因此建议扩展为单仓库双层结构：

- `app/`：后端 FastAPI 应用
- `web/`：前台 Vue 应用
- `admin/`：后台 Vue 应用

这样拆分的原因是前台和后台业务边界清晰、权限模型不同、页面风格和路由组织差异较大，分开管理更利于后续维护与部署。

### 2. 后端分层结构

后端建议按职责拆分：

- `app/main.py`：应用启动入口、注册路由、中间件
- `app/core/`：配置、鉴权、权限依赖、数据库初始化
- `app/models/`：SQLAlchemy 数据模型
- `app/schemas/`：Pydantic 请求与响应模型
- `app/api/`：按前台与后台拆分接口
- `app/services/`：文章、媒体、评论、用户等业务逻辑
- `app/utils/`：通用工具，如 Markdown 渲染、分页、文件处理
- `app/static/uploads/`：媒体上传目录
- `app/tests/`：后端单元测试或接口测试

### 3. 前端结构

前端建议都使用 Vue，但分为两个应用：

- `web/`：公开博客站点
- `admin/`：后台管理台

每个前端应用都至少包含：

- `src/router/`：路由
- `src/views/`：页面
- `src/components/`：组件
- `src/api/`：接口请求封装
- `src/stores/`：状态管理
- `src/assets/`：静态资源

## 五、权限与认证策略

### 1. 认证

使用 JWT 认证，后端统一签发访问令牌。后台接口必须鉴权，前台公开接口按需开放。前台用户登录评论和后台用户登录管理可以共用认证体系，但接口权限边界必须分开。

### 2. 授权

采用 RBAC 模型：

- `reader`：浏览、登录评论、管理自己的个人信息
- `author`：创建文章、编辑自己的草稿与待审文章、管理自己的媒体、查看自己文章评论
- `admin`：审核发布文章、管理全站文章、用户、媒体、评论、站点配置

### 3. 数据级权限控制

必须在服务层或路由依赖中执行，不仅是菜单控制。

关键规则：

- 作者只能操作自己的文章
- 作者提交后不能自行将文章置为 `published`
- 管理员可审核作者文章
- 管理员创建的文章可直接发布
- 评论审核权属于管理员
- 媒体库可按“仅本人”或“全站可见”策略控制，初期建议后台管理员可见全站，作者默认仅见自己上传的媒体

## 八、Markdown 与内容渲染策略

### 1. 编辑侧
后台使用 Vue Markdown 编辑器组件，支持插入图片、音频、视频链接。

### 2. 存储侧
后端保存 Markdown 原文，并在保存或发布时渲染 HTML 存入数据库。

### 3. 展示侧
前台文章详情页直接使用经过后端处理的 HTML 内容，避免每次前端重新渲染 Markdown。

## 九、媒体处理策略

### 1. 存储方式
初期采用服务器本地存储，目录位于后端应用下固定上传目录。

### 2. 文件访问
通过 FastAPI 暴露静态资源访问路径。

### 3. 缩略图
首期仅对图片生成缩略图；音频和视频使用默认类型封面或图标占位。

### 4. 分类方式
数据库树形目录作为逻辑分类，不强制要求和磁盘目录完全一致，以降低后续目录变更成本。

## 十、评论策略

### 1. 评论来源
- 匿名评论
- 登录用户评论

### 2. 审核策略
建议初期统一进入待审核，再由管理员审核通过后展示。这样更安全，也更适合匿名评论场景。

### 3. 评论层级
可先支持单层回复结构，即允许 `parent_id`，但前台首版只需做到基础回复展示即可。

## 十一、数据库与部署策略

### 1. 数据库
首期使用 SQLite，适合个人博客和 2核2G 环境。

### 2. 迁移
需要补充数据库迁移方案，否则模型调整会越来越难维护。建议引入迁移工具。

### 3. 部署
Linux 环境中，后端可通过 `uvicorn` 或进程管理方式运行；前端打包后作为静态资源部署。因为当前是前后端分离，所以可分别部署，也可由反向代理统一入口。

## 十四、实施阶段建议

建议按阶段推进，而不是一次性把全部功能堆完。

### 第二阶段：文章、分类、标签、审核流
目标是打通核心内容管理能力。

### 第三阶段：媒体库
目标是完成上传、树形目录、缩略图、批量操作基础能力。

### 第四阶段：评论系统与前台公开接口
目标是支撑前台博客完整内容消费。

### 第五阶段：Vue 前台与后台页面
目标是把所有后端能力真正接成可用界面。

## 十五、错误处理策略
